# Pre-Registration: Levered SPY with Drawdown Brake (v1)

**Date frozen:** 2026-06-10 (registration = the git commit introducing this file;
results are only valid if generated by code matching this spec, after that commit)

**Objective (user-stated, 2026-06-10):** more terminal wealth than SPY buy-and-hold,
subject to: maximum drawdown tolerance **−25%**, maximum leverage **1.3x**.

**Scope:** SPY only. The sector basket is explicitly out of scope. This document
supersedes all prior backtest claims, all of which were generated by a harness with
one-day look-ahead bias (see `project_lookahead_bias_backtest` memory; fixed via the
`--lag` flag in `backtest/sector_backtest.py`).

---

## 1. Hypothesis under test

At >1x baseline exposure, a crude and infrequent de-risking rule can be net-positive
(it was net-negative at 1x: −0.65% median per round trip, 170 trips, 2010–2025),
because the leverage raises both the payoff of avoided crashes and the cost of not
avoiding them. The leverage supplies the outperformance; the brake only caps the tail.

## 2. Candidate strategies (frozen — no other variants may be evaluated)

All are daily exposure rules on SPY. No parameter below may be changed after
registration. Parameters come from the literature or the user's constraints, not from
fitting.

| ID | Rule | Exposure |
|----|------|----------|
| S0 | SPY buy & hold (wealth benchmark; exempt from DD gate) | 1.0 always |
| S1 | Faber null: above/below 200-day SMA | 1.0 / 0.0 |
| S2 | Levered trend, hard brake | 1.3 / 0.0 |
| S3 | Levered trend, soft brake | 1.3 / 0.65 |
| S4 | Vol target: `clip(0.15 / vol20, 0, 1.3)`; vol20 = std of last 20 daily log returns × √252 | 0–1.3 |
| S5 | Combo: S4's exposure above the 200-day SMA, 0.0 below | 0–1.3 / 0.0 |

No-trade band: a new target exposure is acted on only if it differs from current
exposure by ≥ 0.10 (limits toll frequency for S4/S5; S1–S3 are binary regardless).

## 3. Execution and cost model

- **Timing (the honest convention):** signals computed from close[t]; the exposure
  decided at t earns the close[t]→close[t+1] return. Identical to `--lag 1`.
- **Slippage:** `estimate_slippage(VIX[t]) × |Δexposure|` on every trade, same
  function as the existing harness.
- **Financing:** daily accrual of `(DTB3[t] + 0.40%) / 252 × max(0, exposure − 1)`.
- **Cash yield:** idle cash earns `DTB3[t]` (implementable via T-bills/SGOV).
- **Dividends:** included via adjusted close.
- **Data:** `backtest/data/spy_daily.csv` (1993→), `features_daily.csv` for VIX;
  `DTB3` added to `download_features.py` from FRED. Evaluation window: first date
  all inputs exist (~1994-01) through the latest common date. Reported per decade.

## 4. Pass gates (ALL binding, full sample, net of all costs)

1. **maxDD ≥ −20%** on daily closes. (Buffer below the −25% tolerance: live
   drawdowns routinely exceed backtest drawdowns. −25% in backtest would mean
   planning to violate the tolerance.)
2. **CAGR > S0 (B&H) CAGR.** The point is terminal wealth.
3. **CAGR > S1 (Faber null) CAGR.** Anything levered/complex must beat the dumbest
   public rule, or the complexity isn't earning its risk.

**Winner:** highest CAGR among passers. Tie-break (within 0.25pp): fewer trades.
**If no candidate passes: the program's answer is NO.** Adopt B&H with a written
risk policy; SPY-timing research ends. This sentence is the kill criterion and was
agreed before any result existed.

**Fragility flag (binding):** if the winner's CAGR edge over S0 is negative in the
2010→present sub-period, the paper-trade gate below extends from 6 to 12 months.

## 5. Validity checks (run and pass BEFORE results are read)

- **Canary A (look-ahead sensitivity):** S2 run at lag-0 must beat S2 at lag-1.
  Confirms the harness still detects the inflation we are guarding against.
- **Canary B (no free lunch):** 100 seeded random daily exposures ∈ {0, 1.3} must
  produce median CAGR < S0 net of costs.
- **Spec diff:** the evaluation script (`backtest/prereg_eval.py`, to be written)
  is reviewed against this document before results are examined.

## 6. Anti-snooping rules

- One evaluation run. Results stand whether they flatter or embarrass.
- The only permitted amendment class is a **bug** (code ≠ this spec), documented in
  an AMENDMENT section with the diff. Performance-motivated changes are prohibited.
- Any new strategy idea requires a fresh pre-registration and may not present this
  sample as out-of-sample.
- Sub-period tables are diagnostics; gates are evaluated on the full sample only.
  No post-hoc window selection.

## 7. Paper-trade gate (before any real dollar changes exposure)

Survivor runs ≥ 6 months (12 if fragility-flagged) in shadow alongside the live
System control. Promotion requires: zero spec violations, and realized behavior
consistent with the modeled costs (median daily |simulated − shadow| < 0.05%).
It is explicitly NOT judged on raw outperformance over the window — six months is
noise, and pretending otherwise invites goalpost-moving in both directions.

## 8. Registered predictions (calibration on record, 2026-06-10)

- P(at least one candidate passes all three gates): **45%**
- Most likely passer: S2 or S5.
- P(S4 passes): **15%** (vol-managed portfolios are historically fragile after costs).
- P(program ends in the kill criterion / Road 3): **55%**
